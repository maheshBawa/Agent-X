# Post-Project Self-Reflection

Run this audit after every completed project.

## Questions to Answer

### Effectiveness
1. Did the product meet the user's requirements as specified in the product spec?
2. Were there requirements that were unclear or missing from the spec?
3. Did the architecture hold up during implementation, or were changes needed?

### Quality
4. Did any quality gates catch real issues? Which gates were most valuable?
5. Did any quality gates produce false positives? How many?
6. Were there any issues that slipped through ALL gates?
7. What was the final test coverage percentage?

### Stack Decision
8. Was the tech stack recommendation appropriate for this project?
9. Were there any stack-related challenges during development?
10. Would a different stack have been better in hindsight?

### Process
11. Which phase took the longest? Why?
12. Were there any bottlenecks or repeated failures?
13. Did the user have to correct Agent-X on anything?
14. Were milestone checkpoints at the right frequency?

### Memory & Learning
15. What new preferences did we learn about the user?
16. What patterns should be remembered for future projects?
17. Were past learnings (from memory) applied correctly?

## Output
For each answer, determine:
- **Insight:** What was learned
- **Action:** None / Update memory / File GitHub issue / Fix directly
- **Priority:** Low / Medium / High

Log all insights to `core/evolution/insights.md`.
Create GitHub issues for High and Medium priority improvements.
